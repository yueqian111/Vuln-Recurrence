# Sqli-Labs Less-24 二次注入复现

#### 核心原理

二次注入是存储型注入的一种：先将构造的恶意数据存入数据库，当后续程序读取并拼接该数据到SQL语句时，恶意 payload 被执行。Less-24 的关键在于：

- 注册用户名时未做过滤，直接存入数据库。

- 修改密码时，SESSION 中的用户名被直接拼接到 SQL 更新语句中。

---

### 复现步骤

#### 1. 注册恶意账号

1. 访问 Less-24 的注册页面（如 `http://127.0.0.1/sqli-labs/Less-24/new_user.php`）。

2. 输入以下信息：

    - 用户名：`admin'#`（`'` 用于闭合SQL引号，`#` 用于注释后续语句）

    - 密码：`123456`

    - 确认密码：`123456`

3. 点击注册，完成账号创建。此时数据库中新增一条用户记录，用户名为 `admin'#`。

#### 2. 登录恶意账号

1. 访问登录页面，使用刚注册的账号登录：

    - 用户名：`admin'#`

    - 密码：`123456`

2. 登录成功后，服务器SESSION中存储的 `username` 即为 `admin'#`。

#### 3. 执行二次注入（修改admin密码）

1. 进入修改密码页面（`pass_change.php`）。

2. 输入以下信息：

    - 当前密码：`123456`（恶意账号的原密码，用于通过验证）

    - 新密码：`654321`（任意设置，作为admin的新密码）

    - 确认新密码：`654321`

3. 点击“修改密码”提交。

此时，后端执行的SQL语句会被恶意 payload 篡改：

```SQL

-- 原始SQL模板
UPDATE users SET PASSWORD='$pass' WHERE username='$username' AND password='$curr_pass'

-- 代入恶意用户名后实际执行的SQL
UPDATE users SET PASSWORD='654321' WHERE username='admin'#' AND password='123456'
```

`'` 闭合了前面的引号，`#` 注释了后续条件，最终等价于：

```SQL

UPDATE users SET PASSWORD='654321' WHERE username='admin'
```

#### 4. 验证注入结果

1. 退出当前恶意账号。

2. 使用用户名 `admin` 和新密码 `654321` 尝试登录。

3. 若登录成功，说明二次注入成功，已成功修改admin用户的密码。

### 关键注意事项

- **注释符选择**：MySQL中 `#`、`-- `（后接空格）均可作为行注释，这里使用 `#` 更简洁。

- **前置条件**：恶意账号必须能正常登录，才能进入修改密码环节。

- **环境依赖**：确保sqli-labs Less-24使用MySQL数据库，且未对SESSION中的用户名做二次过滤。
